               <p class="pw-post-body-paragraph my mz gr na b nb nc nd ne nf ng nh ni nj nk nl nm nn no np nq nr ns nt nu nv gk bh" id="9af9">
                Always remember to
                <strong class="na gs">
                 include headers while scraping https sites
                </strong>
                as without it you will have
                <strong class="na gs">
                 access denied
                </strong>
                error. And the headers are same for all so don’t worry. Most https sites don’t allow web scraping, but the point is how were you able to view the website though it is https site ? It is because our Web browsers uses
                <strong class="na gs">
                 headers
                </strong>
                to let the servers know that the request is through a Web browser and so we pass headers to the server.
               </p>
               <pre class="pn po pp pq pr qd qe qf qg av qh bh"><span class="qi ny gr qe b iq qj qk l jf ql" id="0833">url=’<a class="af nw" href="https://www.accuweather.com/en/in/surat/202441/daily-weather-forecast/202441'" rel="noopener ugc nofollow" target="_blank">https://www.accuweather.com/en/in/surat/202441/daily-weather-forecast/202441'</a></span><span class="qi ny gr qe b iq qm qk l jf ql" id="ddaa">headers = {‘User-Agent’: ‘Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/83.0.4103.61 Safari/537.36’}</span></pre>

                <a class="af nw" href="https://pypi.org/project/html5lib/" rel="noopener ugc nofollow" target="_blank">

                </a>
                is a pure python library for parsing html.
               </p>
               <pre class="pn po pp pq pr qd qe qf qg av qh bh"><span class="qi ny gr qe b iq qj qk l jf ql" id="1b1a">r=requests.get(url, headers=headers)<br/>soup1 = BeautifulSoup(r.content, ‘html5lib’)</span></pre>

                  .findAll()
                 </strong>
                </a>

                  .find()
